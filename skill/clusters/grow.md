# 📈 GROW — GTM, Token, Community & Fundraising

Owns attention, distribution, and capital. The cluster most likely to be run prematurely — it
inherits gates from SECURE and OPERATE. Current as of 2026-06.

## Scope

- **GTM:** positioning, launch narrative, channels, growth analytics/funnels.
- **Token:** TGE planning, airdrop/eligibility design, distribution/listing, post-launch comms. (Mechanics build in BUILD; legality/tax in OPERATE; claim safety in SECURE.)
- **Community:** ecosystem presence, content, ambassador/quest programs.
- **Fundraising / hackathons:** narrative, deck, demo, target list, grant/track fit.

## Decide

| If the task is… | Then |
|---|---|
| Idea validation, hackathon/grant prep, startup research | Delegate to `colosseum` |
| Idea→launch journey, GTM datasets | Delegate to `solana-new` |
| Token-launch mechanics (bonding curve, launchpad) | BUILD + delegate to `sendai` (pumpfun, launchlab) |
| Fundraising readiness (entity, SAFT, data room) | Coordinate with **OPERATE** ([clusters/operate.md](operate.md)) |

## Grow discipline

- **No hype, no unverifiable claims.** Avoid price/return promises; avoid attributing quotes to real people.
- A **token launch is never "just GROW"** — it carries a SECURE gate (audit + drainer-safe claim) and an OPERATE gate (securities/tax posture). Surface both ([orchestration.md](../references/orchestration.md)).
- Respect the lifecycle: don't run a TGE before mainnet stability ([company-lifecycle.md](../references/company-lifecycle.md)).

## Verification gate

- Demo works on devnet/mainnet (for raises/hackathons).
- For a token: SECURE gate (audit + claim UX) and OPERATE gate (legal/tax) **passed** before public launch.
- Claims are accurate and substantiated.
